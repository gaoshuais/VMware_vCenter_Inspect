短期 (立即处理 · 严重 0)

中期 (1-2 周内 · 警告 5)

  • [时间同步]Timesync 模式 = DISABLED
    推荐 NTP 模式; HOST/DISABLED 在跨集群迁移或主机时间漂移时可能引发故障
  • [DNS]DNS hostname 仍是 'localhost'
    建议为 VCSA 配置正式 FQDN, 否则证书 / SSO / 备份恢复均可能异常
  • [访问]SSH 已启用
    生产环境建议默认关闭, 排错时临时打开后及时关闭
  • [备份]未配置任何备份计划
    同上
  • [集群]Cluster 'Cloud' 未启用 HA
    ≥ 2 节点集群推荐启用 HA 保障 VM 故障迁移

长期 (持续改进 · 提示 4)

  • [DNS]DNS 服务器数 = 1
    生产环境建议 ≥ 2 个 DNS 服务器避免单点
  • [访问]DCUI 已启用
    DCUI 用于本地控制台紧急恢复, 远程运维场景可关闭
  • [证书]当前 vCenter TLS 证书为自签 (VMSCA / localhost 默认 CA)
    生产可考虑替换为企业 CA 或公共 CA 签发的证书
  • [集群]Cluster 'Cloud' 未启用 DRS
    推荐启用 DRS 自动均衡负载 (FullyAutomated 或 PartiallyAutomated)

17免责声明disclaimer

关于本报告

本报告由 vcenter_inspect.ps1 v1.0 通过 vCenter REST API (vSphere Automation API) 自动采集生成, 不修改任何 vCenter 配置, 不写入任何文件到 vCenter Appliance。

REST API 8.0 限制:

  • 单 ESXi 主机详细信息 (CPU/Memory/Build/Uptime/Maintenance Mode) 已 deprecated, 需用 PowerCLI 或 vSphere SDK 补充。
  • VM 快照列表 / 大小、Alarm / Event / 告警历史、License 状态在 8.0 REST 未暴露, 需 vmodl SOAP 接口。
  • 性能数据 (CPU/Memory/IOPS 历史曲线) 在 stats API (preview), 字段不稳定, 本版本未采集。

建议结合 VAMI (https://) 与 vSphere Client 手工核对告警明细。本报告中的"总体建议"基于通用最佳实践与本次数据生成, 实际优先级请根据业务场景判断。
